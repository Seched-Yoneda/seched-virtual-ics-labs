# プロジェクト用語規約

このプロジェクトのドキュメント作成において、以下の用語の使い分けを厳守する。

## 1. ネットワーク機器の呼称
- **アダプタ**: ハードウェア寄り、または VirtualBox 等の仮想化ソフトウェア側の設定を指す場合にのみ使用する。
  - 例：アダプタ 1、ホストオンリーアダプタ
  - 表記：末尾の長音（ー）は付けず、「アダプタ」とする。
- **インターフェース**: OS内部で認識されているデバイス名や、OS側での論理設定を指す場合に使用する。
  - 例：第1インターフェース、インターフェース eth0、インターフェース名
  - 表記： 「インターフェース」とする（「インタフェース」や「ネットワークインターフェース」は使用しない）。

## 2. 簡略化の原則
- 「ネットワークアダプタ」は単独で使用せず **「アダプタ」** と記述する。
- 「ネットワークインターフェース」は単独で使用せず **「インターフェース」** と記述する。

## 3. 管理者権限の表現
- PowerShellを用いた管理者権限での操作を記述する場合は、冗長な表現を避け、以下のように統一する。
  - `PowerShellを管理者権限で開き` → **`管理者PowerShellを開き`**
  - `管理者権限のコマンドで` → **`管理者PowerShellで`**
  - `PowerShell (管理者)` → **`管理者PowerShell`**

## 4. リモート接続関連の表現
- VS Codeの機能（UI）や状態を指す場合は、以下の表記に統一する。
  - `リモートWindows`、`リモートWindow`、`リモート（SSH）ウィンドウ` → **`リモートウィンドウ`**
- 接続アクションそのものを指す場合は、**`リモート接続する`** または **`SSH接続する`** に統一し、「リモートウィンドウに接続する」といった曖昧な表現は避ける。

## 5. 適用範囲
- 今後、このリポジトリ内の `.md` ファイル等を編集する際、常に上記の呼称と表記（長音なし等）を適用する。
